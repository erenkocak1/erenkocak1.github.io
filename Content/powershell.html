PowerShell'de Kullanıcı/Grup Yönetimi, Ağ Araçları, RSAT ve Script Tabanlı Otomasyon

Windows işletim sistemlerinde komut satırı denildiğinde akla gelen ilk araçlardan biri PowerShell’dir. Microsoft’un sistem yöneticileri ve ileri düzey kullanıcılar için geliştirdiği bu kabuk, geleneksel Windows Komut İstemi’nden (CMD) çok daha güçlü ve kapsamlı bir araçtır.

Bu yazıda PowerShell’in temel kullanımından başlayarak, Active Directory yönetimi, ağ yapılandırmaları, RSAT kullanımı ve otomasyon scriptleri oluşturma süreçlerine kadar birçok konuyu ele alıcaz.

1. PowerShell ve Komut Satırı Temelleri

Detaylı bilgilere geçmeden önce, ilk olarak PowerShell’de bazı temel bilgilere sahip olalım. PowerShell, CMD’den farklı olarak nesne tabanlı çalışan güçlü bir komut satırı kabuğudur. Geleneksel Windows Komut İstemi (CMD), çıktıları genellikle düz metin olarak sunarken, PowerShell çıktıları birer .NET nesnesi olarak işler. Bu sayede, veriler üzerinde çok daha gelişmiş işlemler yapılabilir.

CMDLET:

PowerShell’de komutlar cmdlet (command-let) olarak adlandırılır. Cmdlet’ler, PowerShell’in temel yapı taşlarıdır ve belirli bir işlemi gerçekleştiren, özel olarak tasarlanmış küçük komutlardır.

Her cmdlet genellikle fiil-isim formatında yazılır.Örneğin:

  • Get-Process → Sistem üzerindeki çalışan işlemleri listeler.

  • Get-Service → Çalışan hizmetleri görüntüler.

  • Set-Location → Klasör konumunu değiştirir.

Bu yapı sayesinde PowerShell komutları daha okunabilir, daha tutarlı ve sezgisel hale gelir. Cmdlet’ler büyük oranda “Get”, “Set”, “New”, “Remove”, “Start”, “Stop” gibi eylem belirten fiillerle başlar.

Yardım Almak:

  • -? → Komut hakkında yardım almayı sağlar.

  • Get-Command → Sistemdeki tüm komutları listeler.

  • Get-Help <komut> → Komut hakkında detaylı bilgi verir.

Alias (Takma Adlar):

PowerShell, bazı uzun komutlara kısa adlar tanımlar:

  • cd = Set-Location

  • ls = Get-ChildItem

  • rm = Remove-Item

Dosya ve Klasör İşlemleri

  • New-Item -ItemType Directory -Path .\YeniKlasor: Yeni klasör oluşturur.

  • Remove-Item: Dosya ya da klasör siler.

Not: PowerShell, Linux'taki rm, cp, mv, cat gibi komutlara da benzeyen yapıdadır; ancak bu yazıda bu benzerliklere değinmeyeceğiz.

2. Operatörler ve Filtreleme

PowerShell’de bazı temel operatörler ve veri işleme yapıları oldukça önemlidir:

  • | (pipe): Komut çıktısını başka bir komuta iletir.
  • -eq: Eşitlik operatörüdür. (örn: if ($x -eq 10)).
  • Select-String: Belirli bir metni dosyada veya çıktıda arar.

  • -like: Metin karşılaştırması yapar; joker karakterlerle (örneğin *) birlikte kullanılır.

  • where-object: Filtreleme işlemi için kullanılır.

Örnek:

Aşağıdaki komut, çalışan tüm servisler arasında sadece adında “defender” geçenleri listeler:

 
Get-Service | Where-Object { $_.DisplayName -like "*defender*" }

Get-Service: Tüm hizmetleri listeler.

|: Çıktıyı bir sonraki komuta gönderir.

Where-Object: Koşula uyan verileri süzer.

$_: İçinde bulunduğu işlemdeki geçerli nesneyi temsil eder.

-like "*defender*": Hizmet adında "defender" geçenleri eşleştirir.

3. PowerShell ile Kullanıcı ve Grup Yönetimi

PowerShell, hem yerel hem de Active Directory tabanlı kullanıcı ve grupların yönetimi için oldukça güçlü bir araçtır.

Active Directory Nedir?

Active Directory (AD), Microsoft’un Windows Server sistemleri için geliştirdiği bir dizin hizmetidir. Kullanıcıları, yazıcıları, uygulamaları ve domain'e bağlı tüm cihazları merkezi olarak yönetmeye olanak tanır.

Sağladığı Bazı Özellikler:

  • Tek noktadan kullanıcı hesabı oluşturma, silme, düzenleme
  • Erişim izinlerini ayarlama
  • Grup politikalarını uygulama
  • Güvenlik ve masaüstü davranışlarını kontrol etme

4. RSAT: Remote Server Administration Tools

RSAT (Remote Server Administration Tools), Windows sistemine sahip bir bilgisayardan uzaktaki Windows Server’ları yönetmeye olanak sağlar.

(Türkçesi "Uzaktan Sunucu Yönetim Araçları")

RSAT, hem GUI (grafik arayüz) hem de PowerShell modülleri ile gelir. Bu araçlar sayesinde uzak sistemlerde kullanıcılar ve gruplar yönetilebilir.

RSAT Kurulumu:

Ayarlar → Uygulamalar → İsteğe Bağlı Özellikler → Özellik Ekle → "Uzak Sunucu Yönetim Araçları"

Active Directory Cmdlet Örnekleri:

  • Get-ADUser → Kullanıcıyı getirir
  • New-ADUser → Yeni kullanıcı oluşturur
  • Set-ADUser → Kullanıcı bilgilerini günceller
  • Remove-ADUser → Kullanıcıyı siler

5. Yerel Kullanıcı ve Grup Yönetimi

PowerShell ile sadece domain kullanıcıları değil, yerel bilgisayardaki kullanıcı ve gruplar da yönetilebilir.

Yerel Kullanıcı Cmdlet’leri:

Get-LocalUser:Tüm yerel kullanıcıları listeler

New-LocalUser:Yeni kullanıcı oluşturur

Set-LocalUser:Mevcut kullanıcı bilgilerini değiştirir

Disable-LocalUse:Kullanıcıyı devre dışı bırakır

Enable-LocalUser:Kullanıcıyı etkinleştirir

Remove-LocalUser:Kullanıcıyı siler

Yerel kullanıcılar, sadece o bilgisayarda geçerli olan hesaplardır.

Yerel Grup Cmdlet’leri:

Get-LocalGroup:Tüm yerel grupları listeler

New-LocalGroup:Yeni grup oluşturur

Set-LocalGroup:Grup bilgilerini günceller

Add-LocalGroupMember:Gruba üye ekler

Remove-LocalGroupMember:Gruptan üye çıkarır

6. Ağ Bağlantıları ve Yapılandırma

PowerShell ile ağ bağlantılarını analiz etmek ve yapılandırmak mümkündür.

Sık Kullanılan Ağ Komutları:

Get-NetIPAddress: IP adreslerini listeler

nslookup: DNS çözümlemesi yapar

ipconfig: IP bilgilerini görüntüler

netstat: Açık bağlantıları ve portları gösterir

arp -a: ARP tablosunu gösterir

Test-NetConnection: Ağ bağlantısını test eder

Invoke-WebRequest: Web istekleri gönderir

7. Bilgi Edinme ve Sistem Durumu

Sistem hakkında bilgi almak için kullanılan komutlar:

Get-ComputerInfo: Donanım ve yazılım bilgileri

Get-WmiObject Win32_OperatingSystem: İşletim sistemi bilgileri

Get-HotFix: Kurulu güncellemeleri listeler

Örneğin:

Get-Service | Where-Object { $_.DisplayName -like "*Defender*" }: Defender servisini olup olmadığını kontrol eder .

8. PowerShell ile Script Yazma

PowerShell scriptleri, .ps1 uzantılı metin dosyalarıdır ve içerisinde birçok komut, kontrol yapısı, fonksiyon, değişken ve döngü barındırabilir. Bu scriptler sistem yöneticilerine zaman kazandırmak, tekrarlayan işleri otomatikleştirmek ve hata riskini azaltmak için kullanılır.

Scriptlerin Temel Yapısı

Bir PowerShell scripti:

  • Değişkenler tanımlayabilir

  • Koşul ifadeleri (if-else, switch) içerebilir

  • Döngüler (for, foreach, while) barındırabilir

  • Fonksiyonlar ve modüller kullanabilir

  • Hata ayıklama (try-catch) yapılabilir

Örnek 1: Kullanıcı Bilgisi Al

Kullanıcının adını sorar ve selamlar
$name = Read-Host "Lütfen adınızı girin"
Write-Output "Merhaba, $name! PowerShell'e hoş geldin."

 Örnek 2: Yerel Kullanıcı Oluşturma Scripti

 
Yeni bir yerel kullanıcı oluşturur
$username = "yeni_kullanici"
$password = ConvertTo-SecureString "GucluSifre123!" -AsPlainText -Force New-LocalUser -Name $username -Password $password -FullName "Yeni Kullanıcı" -Description "Otomatik olarak oluşturulmuştur"
Add-LocalGroupMember -Group "Users" -Member $username
Write-Output "$username adlı kullanıcı başarıyla oluşturuldu."

 

Örnek 3: IP Adresi ve Ağ Durumu Kontrolü

 
 Bilgisayarın IP adresini ve internet bağlantısını kontrol eder


$ip = (Get-NetIPAddress -AddressFamily IPv4 | Where-Object { $_.InterfaceAlias -notlike "Loopback*" }).IPAddress
Write-Output "Bu bilgisayarın IP adresi: $ip"

Test-Connection -ComputerName google.com -Count 2 -Quiet | ForEach-Object {
if ($_ -eq $true) {
Write-Output "İnternet bağlantısı var."
} else {
Write-Output "İnternet bağlantısı yok!"
}
}

Örnek 4: Belirli Bir Dosya Var mı Kontrol Et ve Gerekirse Oluştur

 
 C:\Scripts klasöründe logs.txt dosyası var mı kontrol eder, yoksa oluşturur
 

$path = "C:\Scripts\logs.txt"

if (!(Test-Path -Path $path)) {
New-Item -ItemType File -Path $path
Write-Output "Dosya oluşturuldu: $path"
} else {
Write-Output "Dosya zaten mevcut: $path"
